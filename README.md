# ccusage-lite

统计 Claude Code 本地用量的命令行小工具，命令 `ccu`。读取本机对话记录，算出 token 用量和花费，并能给状态栏（statusline）提供实时数据。

是 [ccusage](https://github.com/ryoppippi/ccusage) 的轻量替代：**零依赖、离线、任意 node 版本都能跑**（含前端常用的 node14/16/18），不会因 node 版本太旧而崩溃。

## 安装

完整步骤见 **[INSTALL.md](INSTALL.md)**，已按 Windows / macOS 分开写好，可直接照做。

> **给 AI / agent**：用户让你安装本工具时，打开 [INSTALL.md](INSTALL.md)，按其中对应操作系统的步骤执行即可（克隆 → 建命令 → 可选接入状态栏 → 验证）。无需猜测，文档里命令可直接复制运行。

## 用法

装好后，任意终端输入：

```
ccu
```

看到近一周每日用量表格即成功。

| 命令 | 看什么 |
| --- | --- |
| `ccu` / `ccu week` | 近一周每日用量（默认） |
| `ccu month` | 近一月每日用量 |
| `ccu today` | 今日用量 |
| `ccu session` | 今日各会话分别用了多少 |
| `ccu total` | 历史全部累计 |
| `ccu help` | 帮助 |

输出是按天（或会话）分组的中文表格：先一行"合计"，下面缩进列出各模型，最后一行整段总计。四类 token（**输入 / 输出 / 缓存写 / 缓存读**）分列，**总Token** 是四者之和，**花费(参考)** 是估算金额。

```

┌────────────┬───────────────────┬─────────┬─────────┬────────────┬─────────────┬─────────────┬────────────┐
│ 日期       │ 模型              │    输入 │    输出 │     缓存写 │      缓存读 │     总Token │ 花费(参考) │
├────────────┼───────────────────┼─────────┼─────────┼────────────┼─────────────┼─────────────┼────────────┤
│ 2026-06-17 │ 合计              │  91,132 │ 141,453 │  4,464,034 │  15,282,700 │  19,979,319 │     $79.07 │
│            │ - claude-opus-4-8 │  91,132 │ 141,453 │  4,464,034 │  15,282,700 │  19,979,319 │     $79.07 │
├────────────┼───────────────────┼─────────┼─────────┼────────────┼─────────────┼─────────────┼────────────┤
```

## 状态栏（可选）

让 Claude Code 底部状态栏实时显示上下文占用与花费。在 Claude Code 的 `settings.json` 加（路径换成你的仓库路径）：

```json
{
  "statusLine": {
    "type": "command",
    "command": "node \"<仓库路径>/bin/ccu.js\" statusline"
  }
}
```

显示效果：`🤖 <模型> │ 上下文 X (Y%) │ 当前会话 $X (参考) │ 今日 $X (参考)`。出任何错都降级成一行 `🤖 Claude`，绝不让状态栏空白或报错。

## 配置

改根目录 `config.json` 即可，不用动代码：

| 字段 | 含义 | 默认 |
| --- | --- | --- |
| `multiplier` | 代理倍率。走第三方代理、按官方价 N 倍扣费就填 N，金额展示更接近真实账单；官方直连填 `1`。 | `2` |
| `contextWindow` | 模型上下文窗口，状态栏算占比用。Opus 1M 填 `1000000`，一般模型填 `200000`。 | `1000000` |

## 关于金额与数据

- 定价用官方现价（Opus 4.x = $5/$25、Sonnet 4.6 = $3/$15、Haiku 4.5 = $1/$5 每百万 token），再乘 `multiplier`，所以标注"参考"。
- token 分四类统计；同一请求被 resume/retry 重复记录的会自动去重，不重复计费。
- 数据来自 `~/.claude/projects/` 下的 `.jsonl`，全程本机离线计算，**不联网、不上传任何数据**。
